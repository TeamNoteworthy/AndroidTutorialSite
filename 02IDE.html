Step 2: Installing Android Studio

Android Studio is Google's official IDE for Android development and includes all you need to start developing Apps for Android, including the Android Studio IDE and Android SDK Tools. Plus, it comes with a bunch of useful features and tools! Of course, there are alternatives to Android Studio such as Eclipse and Xamarin Studio, but we will be solely using Android Studio and highly recommend you to do the same for the purpose of this tutorial. You can download Android Studio by clicking here. Figure 2-1 below shows the basic gist of what you'll see.

Before you set up Android Studio, be sure you have installed Java Development Kit (JDK) 6 or higher since the Java Runtime Environment (JRE) alone is not sufficient (JDK 7 is required when developing for Android 5.0 and higher). To check if you have JDK installed (and which version), open a terminal and type javac -version. If the JDK is not available or the version is lower than 6, go download JDK. Refer to the following instructions below on how to setup Android Studio:

To set up Android Studio on Windows:
  1. Launch the .exe file you just downloaded.
  2. Follow the setup wizard to install Android Studio and any necessary SDK tools.
  3. On some Windows systems, the launcher script does not find where Java is installed. If you encounter this problem, you need to set an environment variable indicating the correct location. Select Start menu -> Computer -> System Properties -> Advanced System Properties. Then open Advanced tab -> Environment Variables and add a new system variable JAVA_HOME that points to your JDK folder, for example C:\Program Files\Java\jdk1.7.0_21.

The individual tools and other SDK packages are saved outside the Android Studio application directory. If you need to access the tools directly, use a terminal to navigate to the location where they are installed. For example: C:\Users\YOUR_USER_NAME\sdk\

To set up Android Studio on Mac OS X:
  1. Launch the .dmg file you just downloaded.
  2. Drag and drop Android Studio into the Applications folder.
  3. Open Android Studio and follow the setup wizard to install any necessary SDK tools.
  4. Depending on your security settings, when you attempt to open Android Studio, you might see a warning that says the package is damaged and should be moved to the trash. If this happens, go to System Preferences -> Security & Privacy and under Allow applications downloaded from , select Anywhere. Then open Android Studio again.

If you need to use the Android SDK tools from a command line, you can access them at: /Users/<user>/Library/Android/sdk/.

To set up Android Studio on Linux:
  1. Unpack the downloaded ZIP file into an appropriate location for your applications.
  2. To launch Android Studio, navigate to the android-studio/bin/ directory in a terminal and execute studio.sh.
  3. You may want to add android-studio/bin/ to your PATH environmental variable so that you can start Android Studio from any directory.
  4. Follow the setup wizard to install any necessary SDK tools.

After installing Android Studio, there may still be a couple of packages you need to install to make your package complete. These packages can be downloaded as needed using the Android SDK Manager. To launch the Android SDK Manager, click the SDK Manager icon in the toolbar in Android Studio. If you still require assistance, please refer to Installing the Android SDK.

  • Get the latest SDK tools

    As a minimum when setting up the Android SDK, you should download the latest tools and Android platform. Figure 2-2 shows the Android SDK Manager.

    1. Open the tools directory and select:
      • Android SDK Tools
      • Android SDK Platform-tools
      • Android SDK Build-tools
        •

    Click on the triangle to the right to have a more expanded view on additional libraries and tools you can install for even more APIs!

  • Get the support library for additional APIs (optional)

    The support library is required for:

    1. Android Wear
    2. Android TV
    3. Google Cast

    It also provides these popular APIs:

    1. Navigation drawer
    2. Swipe views
    3. Backward-compatible action bar

    The Android Support Library provides an extended set of APIs that are compatible with most versions of Android. Open the Extras directory and select:

    1. Android Support Repository
    2. Android Support Library

  • Get Google Play services for even more APIs (optional))

    The Google Play services APIs provide a variety of features and services for your Android apps, such as:

    1. User authentication
    2. Google Maps
    3. Google Cast
    4. Games achievements and leaderboards
    5. And much more

    To develop with Google APIs, you need the Google Play services package: Open the Extras directory and select:

    1. Google Repository
    2. Google Play services

    Note: Google Play services APIs are not available on all Android-powered devices, but are available on all devices with Google Play Store. To use these APIs in the Android emulator, you must also install the the Google APIs system image from the latest Android X.X directory in the SDK Manager.

    Once you've selected all the desired packages, continue to install:

    1. Click Install X packages (X being the number of packages you're installing).

That's it! You are now ready to begin building Android applications. As new tools and packages are released, simply launch the SDK Manager to download the new packages for your SDK.

GitHub Integration

GitHub is a service that allows for easy and powerful team collaboration and code review. Using GitHub allows you to contribute and code projects easily within a team environment. If you've never used GitHub before, we highly suggest that you download GitHub for Windows or GitHub for Mac according to your platform and read the instructions to set it up there. Setting up GitHub for the first time could be a bit tricky, so going to this link would be really useful: Setting Up Git. After installing GitHub, launch Android Studio and set the proper Git tools path for Android Studio. This is done by going to File -> Settings -> Version Control -> Git.

Make sure that the path to the Git executable is correct. Refer to Figure 2-3 if you need help finding it. Also, under Version Control, go to GitHub and modify the fields so that they match up to your GitHub account as shown in Figure 2-4.

After you have set up a repository or are a contributor to an existing repository (repo), you can clone the repo by executing the following command after going to the path or directory you want: git clone [url]. Otherwise, you can use the GitHub application for Windows and simply click the Plus Sign icon at the top left and click Clone, which will then list the available repositories to clone. Click clone and then select the directory you wish to put the repository in. Committing and pushing changes using GitHub for Windows is as simple as simply editing the files in the directory that you cloned, which will then prompt the uncommitted changes to the project. This will show all the uncommitted changes you've made to all of the different files in the repository. You can select the files you want to commit, and then after adding a description of the changes you've done, you're ready to commit! Afterwards, sync your local repository by clicking Sync on the top right.

Structure: Projects and Modules

A few definitions based on concepts in Eclipse:

  • A module in Android Studio is like a project in Eclipse.
  • A project in Android Studio is like a workspace in Eclipse (to be precise, a workspace with interdependent projects).

Basically, a project is an organizational unit that represents a complete software solution. For Android, that means one project per app, and one module per library.

Structure: Folders

Top Level Folders

  1. Main Project

    2. This would be entire project context. Figure 2-3 above shows a top level view of an example project. In Eclipse, this would be your workspace except it's limited to what's relevant to your project. For example, appProject would be your project context if the name of your application was app.

  2. .idea

    3. This is where project specific metadata is stored by Android Studio.

  3. Project Module

    4. This is the actual project. ex: app would be the project module if your app's name was app. Inside the project module has a few other folders which we will detail below:

    • manifests - This is where you put the AndroidManifest.xml. Every application must have an AndroidManifest.xml file (with precisely that name) in its root directory. The manifest file presents essential information about your app to the Android system--information the system must have before it can run any of the app's code.

    • java - This is where you put all of your Java source code that implements the functionality of your app.

    • res - This folder contains various visual resources.

    • drawable - This folder is where you put resources that include images for icons, logos, pictures, etc.

    • layout - This folder is where you place the .xml files that specify each Activity's layout each Activity.

    • menu - This folder is where you place the .xml files that relate to specific menus for certain activities.

    • values - This folder is is intended for storing .xml files that describe resources of various types.

  4. gradle

    5. This is the Gradle build system, which we'll discuss in more detail in the next section.

  5. External Libraries

    6. This is not actually a folder but a place where Referenced Libraries are shown.

What Exactly is Gradle and Maven?

As you've created new projects, you've probably noticed the term Gradle and Maven being thrown out here and there and are unsure of what exactly they do.

Gradle is an open source build automation system. Gradle can automate the building, testing, publishing, deployment, and more of software packages or other types of projects such as generated static websites, generated or documentation.

Maven, on the other hand, is a build management tool. It defines how your .java files get compiled to .class files, packaged into .jar (or .war,.ear) files, pre-processed or post-processed with tools, managing your CLASSPATH, and all others sorts of tasks that are required to build your project. It is similar to Apache Ant or Makefiles in C/C++, but it attempts to be completely self-contained in it that you shouldn't need any additional tools or scripts by incorporating other common tasks like downloading & installing necessary libraries, because it downloads all of the necessary libraries and completes routine tasks for you.

There are three things about Maven that are very beneficial.

  1. After you declare which libraries you are using, Maven will download all the libraries that you use and and the libraries that they use for you automatically. This is very nice, and makes dealing with lots of libraries easy and seamless.
  2. It follows "Convention over Configuration", which basically means by default you don't need to define the tasks you want to do. Unlike Ant or Makefiles, you don't have to write a "compile", "test", "package", or "clean" step. Just put the files in the places Maven expects them and it should work just fine.
  3. Maven also has lots of nice plug-ins that you can install that will handle various routine tasks that may fit your needs.

Of course, fully understanding Gradle and Maven are outside the scope of this tutorial so feel free to learn more about Gradle on your own and Maven as well.
